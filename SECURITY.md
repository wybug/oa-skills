# 环境变量配置指南

## 🔐 安全说明

本技能需要在 CoPaw 中配置 OA 系统的用户名和密码。为确保数据安全，请遵循以下原则：

### ⚠️ 重要安全提示

1. **切勿在对话中透露敏感信息**
   - 不要在对话中直接发送用户名或密码
   - AI助手不会主动询问您的密码
   - 所有敏感信息仅在本地存储

2. **环境变量安全**
   - 环境变量存储在本地配置文件中
   - 不会上传到云端或服务器
   - 仅供本地脚本使用

3. **定期更换密码**
   - 建议定期更换OA系统密码
   - 更换密码后需更新环境变量配置

## 📝 配置步骤

### 步骤1: 打开环境变量配置

1. 在 CoPaw 中进入设置
2. 找到 "Environments" 或 "环境变量" 配置
3. 点击添加新变量

### 步骤2: 添加用户名变量

```
变量名: OA_USER_NAME
变量值: [你的OA用户名]
```

**注意**:
- 用户名通常是你的工号或邮箱前缀
- 不要在对话中透露具体用户名

### 步骤3: 添加密码变量

```
变量名: OA_USER_PASSWD
变量值: [你的OA密码]
```

**注意**:
- 密码区分大小写
- 请确保密码正确
- 不要在对话中透露密码

### 步骤4: （可选）配置状态文件路径

```
变量名: OA_STATE_FILE
变量值: /tmp/oa_login_state.json
```

**说明**:
- 默认值: /tmp/oa_login_state.json
- 可以自定义路径
- 用于存储登录状态

## ✅ 验证配置

配置完成后，可以通过以下方式验证：

```bash
# 方式1: 执行登录脚本
./scripts/login.sh

# 方式2: 检查环境变量（不显示具体值）
if [ -n "$OA_USER_NAME" ] && [ -n "$OA_USER_PASSWD" ]; then
    echo "✅ 环境变量已配置"
    echo "用户名长度: ${#OA_USER_NAME} 字符"
    echo "密码长度: ${#OA_USER_PASSWD} 字符"
else
    echo "❌ 环境变量未配置"
fi
```

## 🛡️ 安全最佳实践

### 1. 密码管理

- ✅ 使用强密码（包含大小写字母、数字、特殊符号）
- ✅ 定期更换密码（建议每3-6个月）
- ✅ 不要与其他系统使用相同密码
- ❌ 不要在对话中透露密码
- ❌ 不要将密码写在纸上或文档中

### 2. 环境变量管理

- ✅ 使用 CoPaw 内置的环境变量功能
- ✅ 定期检查变量是否过期
- ✅ 离职或换岗时删除相关配置
- ❌ 不要通过消息发送敏感信息
- ❌ 不要截图或记录配置界面

### 3. 会话管理

- ✅ 登录状态有效期约30分钟
- ✅ 定期清理状态文件: `rm /tmp/oa_login_state.json`
- ✅ 使用完毕后关闭浏览器会话
- ❌ 不要在公共电脑上保存登录状态

### 4. 日志管理

- ✅ 定期清理日志文件: `/tmp/oa_*.log`
- ✅ 定期清理截图文件: `/tmp/oa_*.png`
- ✅ 检查日志中是否包含敏感信息
- ❌ 不要将日志文件发送给他人

## 🔧 故障排查

### 问题1: 提示环境变量未配置

**检查步骤**:
```bash
# 检查变量是否存在
echo "用户名配置: ${OA_USER_NAME:+已配置}"
echo "密码配置: ${OA_USER_PASSWD:+已配置}"
```

**解决方案**:
1. 确认已在 CoPaw 中配置环境变量
2. 重启 CoPaw 后重试
3. 检查变量名是否正确（区分大小写）

### 问题2: 登录失败

**可能原因**:
- 用户名或密码错误
- 网络连接问题
- OA系统维护中
- 需要验证码（当前不支持）

**解决方案**:
1. 验证用户名和密码是否正确
2. 检查网络连接
3. 联系IT部门确认系统状态

### 问题3: 提示数据安全警告

**说明**:
如果脚本检测到可能的数据安全问题，会显示警告信息。

**解决方案**:
1. 确认环境变量配置正确
2. 不要在对话中透露敏感信息
3. 检查日志文件是否包含敏感数据
