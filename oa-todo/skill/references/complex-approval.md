# 复杂审批流程 - 人工+智能体交互资料审查

**适用场景**：需要深度审查的复杂审批，包含大量文字资料、附件、技术文档，或需要专业判断的审批。

**核心特性**：
- ✅ 智能体主动提取和总结审批资料
- ✅ 交互式审阅：用户指定关注点，智能体深度分析
- ✅ 智能建议：基于分析结果给出审批建议
- ✅ 暂停与恢复：支持暂时终止审批并保存断点

## 智能体复杂审批行为规范

### 1. 识别复杂审批
- 待办包含大量附件、文档、或需要专业判断
- 主动提出使用复杂审批流程

### 2. 创建断点
```bash
oa-todo approve <fdId> --pause --timeout 15
```
- 解析 JSON 输出
- 保存 session 和 fdId 用于后续操作

### 3. 获取页面内容
```bash
# 使用 agent-browser 获取页面快照
npx agent-browser --session <session> snapshot

# 提取页面内容（JavaScript）
npx agent-browser --session <session> eval "document.body.innerHTML"
```
- 提取关键信息：
  - 基本信息：标题、提交人、金额等
  - 主要内容：文字描述、清单等
  - 附件列表：文件名、大小等
  - 识别图片或文字（OCR）

### 4. 展示总结并询问
- 使用 Markdown 格式展示待办资料总结
- 明确展示：标题、类型、提交人、金额、主要内容、附件列表
- 询问用户关注点（支持多选）
- 提供常见关注点建议
- 等待用户明确指定关注点

### 5. 深度分析
- 根据用户指定的关注点进行分析
- 使用 agent-browser 的 eval 功能提取数据
- 调用外部工具或API进行验证
- 进行数据对比和交叉验证
- 识别潜在风险和问题
- 整理分析结果
- 提供明确的审批建议
- 说明建议理由

### 6. 等待用户决策
- 展示分析结果和建议
- 提供三个选项：
  1. 同意（可附带条件）
  2. 驳回
  3. 暂停审批（保存断点）
- 等待用户选择
- 如果选择同意，询问是否需要附带条件
- 如果选择驳回，询问驳回理由
- 如果选择暂停，保存断点供后续处理

### 7. 执行审批
- 根据用户决策执行审批操作
- **使用 CLI 命令执行审批（内部实现）**：`oa-todo approve <fdId> <action> --comment "<意见>"`
- 资料审查使用 agent-browser，但审批动作使用 CLI 命令
- 添加审批意见（如有）
- 确认审批成功
- 断点自动关闭
- 用户看到：自然语言描述的执行动作，不展示 CLI 命令

## 常见关注点建议

### 采购类
- 价格合理性
- 供应商资质
- 技术规格
- 预算
- 交付周期

### 报销类
- 费用明细
- 票据真实性
- 合规性
- 预算

### 合同类
- 条款合理性
- 法律风险
- 履约保证金

### 人事类
- 简历真实性
- 薪资合理性
- 工作经验

### 财务类
- 财务数据
- 票据真实性
- 合规性

### 技术类
- 技术方案
- 风险评估
- 资源需求

## 注意事项

### ⚠️ 超时管理
- 复杂审批流程可能超过10分钟
- 建议使用 `--timeout 15` 或 `20`
- 如果超时，断点会自动清理
- 用户可以在后续会话中恢复断点：`oa-todo approve <fdId> --pause`

### 💡 断点恢复
- 如果断点超时被清理
- 用户可以重新创建断点恢复审批
- 建议在断点有效期内完成审批

### 🔒 数据安全
- 审批涉及敏感信息时，提醒用户注意保密
- 不要在聊天记录中泄露敏感数据
- 审批完成后，相关数据会从浏览器会话中清除

### 📋 记录保存
- 复杂审批建议保存审批意见
- 可选：将分析结果保存到文件中供后续参考
